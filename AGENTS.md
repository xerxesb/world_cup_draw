# AGENTS.md

## Project Overview

This repository contains a single-screen office TV dashboard for a no-money World Cup sweep. It is a Vite + React + TypeScript static app deployed to GitHub Pages.

The dashboard combines:

- Fixed sweep allocations from `public/data/allocations.json`.
- Editable side awards from `public/data/side-awards.json`.
- A generated tournament snapshot from `public/data/live-data.json`.
- Scoring and normalization logic in `src/lib/`.
- Display components in `src/components/`.

## Setup Commands

- Install dependencies: `npm install`
- Start local dev server: `npm run dev`
- Refresh live data snapshot: `npm run refresh:data`
- Run tests once: `npm test -- --run`
- Run production build: `npm run build`
- Preview production build: `npm run preview`

## Verification Before Completion

Run these before claiming work is complete:

```bash
npm test -- --run
npm run build
```

For visual/layout changes, also run the app and verify the dashboard fits without scrolling at:

- `1366x768`
- `1920x1080`

The dashboard is intended to fit one 16:9 display without page scrolling.

## Development Workflow

- Use development superpowers when available.
- For new features or behavior changes, use the brainstorming/planning workflow before implementation.
- For bugs, use systematic debugging before changing code.
- For feature or bugfix code, write failing tests first, then implement the smallest passing change.
- Keep changes scoped to the requested behavior.
- Prefer pure logic in `src/lib/` and display-only React components in `src/components/`.
- Do not leave dev servers running after verification.

## Code Style

- TypeScript strict mode is enabled.
- Use React function components.
- Keep data normalization and scoring deterministic and covered by unit tests.
- Use `lucide-react` for icons when an icon is needed.
- Keep UI text concise enough for the TV layout.
- Do not introduce large UI frameworks unless explicitly requested.
- Do not add comments for obvious code; add short comments only when they clarify non-obvious behavior.

## Data Rules

- `world_cup_sweep_results.csv` is the historical source of the draw.
- `public/data/allocations.json` is the app-consumed draw data. Keep it in sync if the CSV is intentionally changed.
- `public/data/side-awards.json` is manually editable. Set `winner` to a name when an award is decided; leave it `null` while undecided.
- `public/data/live-data.json` is generated by `scripts/refresh-live-data.mjs`.
- Do not hand-edit `public/data/live-data.json` unless repairing an urgent production issue; prefer `npm run refresh:data`.
- The refresh script should try the public API first and keep a raw GitHub fallback.

## Deployment

- GitHub Pages deploys from `.github/workflows/deploy-pages.yml`.
- Scheduled data refresh runs from `.github/workflows/refresh-data.yml`.
- The public site is `https://xerxesb.github.io/world_cup_draw/`.
- Pushing to `main` triggers a Pages deployment.
- Verify the deployed URL after workflow changes.

- Do not revert user changes unless explicitly asked.
- Keep generated or local-only files out of commits: `node_modules/`, `dist/`, `coverage/`, and `artifacts/`.

## Project Structure

- `.github/workflows/`: GitHub Actions for deploy and data refresh.
- `docs/superpowers/`: planning and design notes from the development workflow.
- `public/data/`: app data loaded by the browser.
- `scripts/refresh-live-data.mjs`: live data fetch and normalization script.
- `src/lib/tournament.ts`: tournament normalization, team name matching, and leaderboard scoring.
- `src/lib/data.ts`: dashboard data loading and fallback behavior.
- `src/lib/display.ts`: display-name shortening for tight TV layout.
- `src/components/`: dashboard panels.
- `src/styles.css`: single-screen responsive layout.
